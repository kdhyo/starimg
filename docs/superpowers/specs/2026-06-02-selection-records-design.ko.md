# 선택 기록 페이지 설계

## 목표

월드컵별로 저장된 이미지 선택 기록을 볼 수 있게 하고, 여러 플레이 기록을 선택해 서로 겹치는 이미지와 각 기록에만 있는 이미지를 비교할 수 있게 한다. 사용자는 이 페이지가 새 게임 결과나 전체 인기 순위가 아니라, 과거에 저장된 선택 기록을 보는 곳이라는 점을 바로 이해해야 한다.

## 진입 방식

메인페이지의 각 월드컵 카드에는 이미지와 월드컵 정보 아래에 두 가지 액션을 둔다.

- `시작`
- `선택 기록 보기`

버튼은 이미지 위에 겹쳐 올리지 않고, 카드 하단의 전용 액션 영역에 배치한다. 이렇게 하면 커버 이미지, 제목, 이미지 수가 버튼에 가리지 않는다. `선택 기록 보기`를 누르면 해당 월드컵의 선택 기록 페이지로 이동한다.

## 페이지 구조

선택 기록 페이지는 데스크톱에서는 좌우 2단 구조, 모바일에서는 위아래로 쌓이는 구조를 사용한다.

왼쪽 영역:

- 선택한 월드컵에 저장된 플레이 기록을 보여준다.
- 기록은 `createdAt` 기준 최신순으로 정렬한다.
- 같은 닉네임이 여러 번 저장되어도 합치지 않는다. 같은 닉네임의 기록도 저장된 플레이 기록 단위로 각각 보여준다.
- 각 행은 `닉네임 · 저장일시 · 선택 이미지 수` 형태로 표시한다.
- 체크박스로 여러 기록을 선택할 수 있게 한다.
- 닉네임 검색 입력을 제공한다. 검색은 화면에 보이는 기록만 필터링하며, 검색 때문에 잠시 숨겨진 기록의 선택 상태는 유지한다.

오른쪽 영역:

- 체크된 기록들의 비교 결과를 보여준다.
- 현재 선택된 기록 수를 보여준다.
- 별점 필터를 제공한다.
- 이미지를 명확한 비교 섹션으로 나눠 보여준다.

## 용어

기능명과 진입 버튼에는 `선택 기록`이라는 용어를 일관되게 사용한다.

권장 페이지 제목:

- `선택 기록`

권장 보조 문구:

- `사람별 플레이 기록을 선택해 겹치는 이미지와 각자만 고른 이미지를 비교합니다.`

## 별점 필터

비교는 먼저 각 선택 기록에 별점 필터를 적용한 뒤, 필터링된 이미지 목록을 기준으로 겹침 여부를 계산한다.

필터 옵션:

- `전체 선택`: 해당 기록에서 한 번이라도 선택된 모든 이미지. 기본값이다.
- `최고 별점만`: 해당 기록에서 가장 높은 별점 그룹에 속한 이미지만 사용한다.
- `별 3개 이상`: 별점이 3개 이상인 이미지만 사용한다.

선택한 필터를 적용했을 때 어떤 기록에 이미지가 하나도 없더라도, 그 기록의 선택 상태는 유지한다. 대신 현재 필터에서는 이미지가 없다는 상태를 보여준다.

## 비교 그룹

두 개 이상의 기록이 선택되면 오른쪽 영역에 다음 그룹을 보여준다.

- `모두 겹친 이미지`: 필터 적용 후 선택된 모든 기록에 공통으로 들어 있는 이미지.
- `일부만 겹친 이미지`: 선택된 기록 중 2개 이상에는 있지만, 모든 기록에 있지는 않은 이미지.
- `각 기록에만 있는 이미지`: 정확히 하나의 기록에만 있는 이미지. 기록별로 묶어 보여준다.

기록이 하나만 선택된 경우에는 겹침 라벨 없이 해당 기록의 필터링된 이미지를 보여준다.

기록이 하나도 선택되지 않은 경우에는 왼쪽 목록에서 기록을 선택하라는 안내를 보여준다.

## 데이터 모델

기존 최종 결과 기록은 `data/results.json`에 다음 필드로 저장되어 있다.

- `id`
- `collectionId`
- `collectionName`
- `nickname`
- `results`
- `roundSelections`
- `createdAt`

중간 라운드 다운로드 기록은 `type: "round-selection-download"`를 가진다. 이 기록은 사용자가 중간 라운드 선택 이미지를 다운로드했을 때 생기는 별도 산출물이므로, 선택 기록 페이지에는 노출하지 않는다.

선택 기록 페이지에서 비교 가능한 플레이 기록은 최종 결과 기록만으로 제한한다.

## API 설계

월드컵별 선택 기록 목록 API를 추가한다.

`GET /api/collections/:collectionId/results`

응답 형태:

```json
{
  "results": [
    {
      "id": "result-id",
      "collectionId": "collection-id",
      "collectionName": "스냅",
      "nickname": "하늘",
      "createdAt": "2026-06-02T10:30:00+09:00",
      "results": {
        "5": ["a.jpg"],
        "4": ["b.jpg", "c.jpg"]
      },
      "selectedImageCount": 3
    }
  ]
}
```

서버 동작:

- `data/results.json`에서 읽는다.
- `type` 필드가 있는 기록은 제외한다.
- `collectionId`로 필터링한다.
- `createdAt`을 `Date.parse`로 해석해 최신순으로 정렬한다. 날짜가 유효하지 않은 기록은 유효한 날짜 기록 뒤에 둔다.
- 잘못된 `results` 값은 서버가 실패하지 않도록 빈 그룹으로 정규화한다.
- `selectedImageCount`는 정규화된 `results` 그룹 전체에서 중복을 제거한 이미지 ID 수로 계산한다.

클라이언트는 다음 API도 함께 호출한다.

`GET /api/collections/:collectionId/images`

클라이언트는 저장된 기록의 이미지 ID를 월드컵 이미지 객체와 매칭한다. 이를 통해 비교 섹션에서 `previewUrl`과 `originalUrl`을 사용해 이미지를 렌더링할 수 있다.

## 클라이언트 라우팅

현재 앱은 하나의 React 컴포넌트 안에서 화면 상태를 전환하는 구조다. 이 흐름을 크게 바꾸지 않도록, 다음과 같은 URL 기반 뷰를 가볍게 추가한다.

`/collections/:collectionId/records`

Express 서버는 이미 API가 아닌 경로를 빌드된 프론트엔드로 fallback하고 있으므로, React Router를 꼭 추가하지 않아도 클라이언트에서 이 URL을 처리할 수 있다. 구현 중 자연스럽게 필요해지는 경우에만 라우터 도입을 검토한다.

## UI 동작

메인페이지:

- 월드컵 카드는 게임 시작을 위한 선택 카드 역할을 유지한다.
- `시작` 액션은 해당 월드컵 게임을 시작한다.
- `선택 기록 보기` 액션은 해당 월드컵의 선택 기록 페이지로 이동한다.

선택 기록 페이지:

- 월드컵 메타데이터, 월드컵 이미지 목록, 월드컵 선택 기록을 불러온다.
- 로딩 상태와 빈 상태를 보여준다.
- 최종 결과 기록이 있으면 기본적으로 최신 기록 최대 3개를 선택한다.
- 사용자가 기록을 직접 체크하거나 체크 해제할 수 있다.
- 닉네임으로 보이는 기록을 필터링할 수 있으며, 현재 선택 상태는 유지한다.
- 선택된 기록이나 별점 필터가 바뀔 때마다 비교 그룹을 즉시 다시 계산한다.
- 이미지는 기존 결과 화면의 이미지 모달 패턴을 따라 원본을 확대해서 볼 수 있게 한다.
- 모바일에서는 기록 목록과 비교 결과를 위아래로 쌓고, 간격을 안정적으로 유지한다. 버튼, 필터, 기록 행, 섹션 제목, 이미지 라벨이 서로 겹치거나 잘리거나 좁은 칸 안에서 읽기 어려운 형태로 줄바꿈되면 안 된다.

## 오류 처리

- 월드컵을 찾을 수 없으면 월드컵을 찾을 수 없다는 메시지와 메인으로 돌아가는 액션을 보여준다.
- 기록을 불러오지 못하면 오류 메시지를 보여주고, 메인으로 돌아가는 액션은 계속 노출한다.
- 저장된 기록에는 있지만 현재 월드컵 이미지 폴더에 없는 이미지 ID가 있으면, 파일명을 표시하는 placeholder 카드를 보여준다. 기록 자체는 이해할 수 있게 유지한다.
- 최종 결과 기록이 없으면, 월드컵을 완료한 뒤 기록이 나타난다는 빈 상태를 보여준다.

## 테스트

서버 테스트:

- `GET /api/collections/:collectionId/results`가 해당 월드컵의 최종 결과 기록만 반환한다.
- 결과가 최신순으로 정렬된다.
- `round-selection-download` 기록은 제외된다.
- 잘못되었거나 비어 있는 결과 그룹 때문에 endpoint가 실패하지 않는다.

클라이언트 테스트:

- 메인 월드컵 카드가 기존 시작 흐름을 유지하면서 `선택 기록 보기`를 노출한다.
- `선택 기록 보기`를 클릭하면 해당 월드컵의 선택 기록 페이지가 열린다.
- 기록 목록이 최신순으로 렌더링된다.
- 여러 기록을 선택하면 모두 겹친 이미지, 일부만 겹친 이미지, 각 기록에만 있는 이미지가 구분되어 표시된다.
- 별점 필터를 바꾸면 비교 결과가 다시 계산된다.
- 모바일 UI 검증은 필수다. 예를 들어 390px 폭 같은 좁은 viewport에서 월드컵 카드 액션, 기록 행, 검색/필터 컨트롤, 비교 섹션 제목, 이미지 그리드가 깨지거나 겹치거나 글자가 잘리거나 어색하게 줄바꿈되지 않는지 테스트하거나 수동 검증한다.
- 모바일에서 글자가 줄바꿈되는 경우에는 충분한 세로 공간을 확보한 의도적인 줄바꿈이어야 한다. 긴 닉네임, 저장일시, 라벨은 주변 컨트롤을 가리지 않도록 말줄임, 자연스러운 줄바꿈, 보조 줄 이동 중 하나로 처리한다.

## 제외 범위

- 비교 묶음을 이름 붙여 저장하는 기능.
- 저장된 기록을 수정하거나 삭제하는 기능.
- 모든 사용자 기준 인기 순위 집계.
- 닉네임별 기록 병합.
- 중간 라운드 `round-selection-download` 기록을 비교 대상에 포함하는 기능.
